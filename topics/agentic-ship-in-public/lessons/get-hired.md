# Ship in public — get hired

## Ship in public to get hired

The market for agentic engineers rewards **proof over pedigree**. You do not need permission or a
platform — you need one real agent, shipped in public, with the artifacts that let a stranger verify it.
"Ship in public" means the work is *linkable*: a repo anyone can open, a README anyone can read, a demo
anyone can watch, evals anyone can rerun. Shipping in public is how a portfolio does its job while you
sleep — it argues for you in rooms you are not in.

The thing that gets you hired is **working agents, not a list of skills.** A candidate who links two real
agents — each with a README that defends an architecture decision, an eval suite, and a demo — beats a
candidate whose resume lists "LLMs, RAG, agents, fine-tuning" with nothing to open. The reason is simple:
a working agent is *falsifiable proof*, and a skills list is a claim. Reviewers trust proof. So the move
is to **ship** — take the capstone from this track, put it somewhere public, write the thread, and link
the demo.

Two more shipped agents beat two more buzzwords every time. The eval-harness exercise in this lesson is a
piece of exactly that proof: a tiny reusable harness that runs an agent over cases and reports a pass rate
— the measurement layer you attach to every agent you ship. Build it once, and every future capstone comes
with evidence baked in.
